NBKRIST JAVAHUB - Interfaces: An Overview with Examples

Interfaces in Java are a blueprint of a class. They specify a set of methods that a class must implement. Interfaces are used to achieve abstraction and multiple inheritance in Java. They define a contract, and any class that implements the interface must adhere to that contract by providing concrete implementations for the methods declared in the interface.

1. Declaration of Interface

An interface is declared using the interface keyword. All methods declared in an interface are implicitly public and abstract (before Java 8). Interface variables are implicitly public, static, and final.

Example:

interface Animal {
    void speak();
    void eat();
}

2. Implementation of Interface

A class uses the implements keyword to implement an interface. It must provide an implementation for all the abstract methods declared in the interface.

Example:

class Dog implements Animal {
    public void speak() {
        System.out.println("Dog barks.");
    }
    public void eat() {
        System.out.println("Dog eats bones.");
    }
}

3. Multiple Interfaces

Unlike classes, a class can implement multiple interfaces. This is how Java achieves multiple inheritance of type.

Example:

interface Swimmer {
    void swim();
}

class Duck implements Animal, Swimmer {
    public void speak() {
        System.out.println("Duck quacks.");
    }
    public void eat() {
        System.out.println("Duck eats insects.");
    }
    public void swim() {
        System.out.println("Duck swims in water.");
    }
}

4. Nested Interfaces

An interface can be declared inside a class or another interface. It's a static member of the enclosing class or interface.

Example:

class MyClass {
    interface MyNestedInterface {
        void myMethod();
    }
}

class MySubClass implements MyClass.MyNestedInterface {
    public void myMethod() {
        System.out.println("Nested interface method implemented.");
    }
}

5. Inheritance of Interfaces

An interface can inherit from another interface using the extends keyword. An interface cannot implement another interface.

Example:

interface AquaticAnimal extends Animal {
    void swim();
}

class Fish implements AquaticAnimal {
    public void speak() {
        System.out.println("Fish doesn't speak.");
    }
    public void eat() {
        System.out.println("Fish eats plankton.");
    }
    public void swim() {
        System.out.println("Fish swims with fins.");
    }
}

6. Default Methods in Interfaces

Introduced in Java 8, default methods allow you to add new methods to an interface without breaking existing implementing classes. They are declared using the default keyword.

Example:

interface Greeting {
    void sayHello();
    default void sayGoodbye() {
        System.out.println("Goodbye!");
    }
}

class EnglishSpeaker implements Greeting {
    public void sayHello() {
        System.out.println("Hello!");
    }
}

7. Static Methods in Interface

Also introduced in Java 8, static methods can be defined in an interface. They are similar to static methods in a class and cannot be overridden. They are called using the interface name.

Example:

interface MathOperations {
    static int add(int a, int b) {
        return a + b;
    }
}

class Main {
    public static void main(String[] args) {
        System.out.println("Sum is: " + MathOperations.add(5, 10));
    }
}

8. Functional Interfaces

A functional interface is an interface with exactly one abstract method. They are used to work with Lambda expressions and method references. The @FunctionalInterface annotation is optional but good practice to ensure the interface meets the criteria.

Example:

@FunctionalInterface
interface MyFunctionalInterface {
    void performAction();
}

class MyClass2 {
    public static void main(String[] args) {
        MyFunctionalInterface action = () -> System.out.println("Performing an action.");
        action.performAction();
    }
}

9. Annotations

Annotations provide metadata about a program's code without affecting its execution. They are a form of marker or metadata that can be used by compilers, build tools, or runtime libraries. They are not interfaces in the traditional sense, but they are declared using the @interface keyword.

Example:

import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface MyAnnotation {
    String value() default "Default value";
}

class AnnotationExample {
    @MyAnnotation(value = "Custom message")
    public void myMethod() {
        System.out.println("Method with annotation.");
    }
}
