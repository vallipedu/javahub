What is an Exception in Java?

An Exception is an event that occurs during the execution of a program that disrupts the normal flow of the program's instructions. When an exception occurs, a method can't handle it, and the program's execution is terminated. Exception handling is a mechanism that allows you to manage these runtime errors to maintain the smooth flow of the program.

What's the difference between a Checked Exception and an Unchecked Exception?

  • Checked Exceptions: These are exceptions that are checked at compile time. The compiler forces the programmer to handle these exceptions either by using a try-catch block or by declaring them with the throws keyword. They are usually recoverable errors, such as IOException or SQLException.
  • Unchecked Exceptions: These are exceptions that are not checked at compile time and occur at runtime. The compiler does not force you to handle them. They often indicate a programming logic error, such as trying to access an invalid index in an array (ArrayIndexOutOfBoundsException) or using a null reference (NullPointerException). They are subclasses of RuntimeException.

What are the 5 keywords used for exception handling in Java?

The five keywords used for exception handling are:

  1. try
  2. catch
  3. finally
  4. throw
  5. throws

Explain the purpose of the try, catch, and finally blocks.

  • try block: This block contains the code that might throw an exception. If an exception occurs within this block, it is "thrown" and caught by an associated catch block.
  • catch block: This block handles the exception. It is executed only if the try block throws an exception of the type specified in the catch block's parameter. You can have multiple catch blocks for a single try block to handle different types of exceptions.
  • finally block: This block contains code that must be executed regardless of whether an exception was thrown or not. It's typically used for cleanup operations, such as closing files or database connections.

try {
    // Code that might throw an exception
} catch (Exception e) {
    // Code to handle the exception
} finally {
    // Cleanup code that always runs
}

What is the difference between the throw and throws keywords?

  • throw: This keyword is used to explicitly throw a single exception instance from a method or any block of code. It is followed by an instance of Throwable.
  • throws: This keyword is used in a method's signature to declare which exceptions the method might throw. It is followed by a comma-separated list of exception class names. This is mandatory for checked exceptions.

// Example of 'throws' in a method signature
void readFile() throws IOException {
    // ...
}

// Example of 'throw' inside a method
void validateAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("Age cannot be negative.");
    }
}

Can a try block exist without a catch or finally block?

No. A try block must be followed by at least one catch block or a finally block. It cannot exist on its own. You can have a try-catch, a try-finally, or a try-catch-finally combination.

When is the finally block not executed?

The finally block is almost always executed. The only scenarios where it might not execute are:

  • The program exits early (e.g., by calling System.exit(0)).
  • The program is terminated by a fatal error that the JVM cannot handle, such as a stack overflow or out of memory error.

What are some common types of unchecked exceptions you have encountered?

Common unchecked exceptions include:

  • NullPointerException: Thrown when an application attempts to use null in a case where an object is required.
  • ArrayIndexOutOfBoundsException: Thrown to indicate that an array has been accessed with an illegal index.
  • IllegalArgumentException: Thrown to indicate that a method has been passed an illegal or inappropriate argument.
  • ArithmeticException: Thrown when an exceptional arithmetic condition has occurred, such as division by zero.

What is the purpose of the `printStackTrace()` method?

The e.printStackTrace() method is a debugging tool used to print a stack trace to the standard error stream. A stack trace provides a detailed list of methods that were called, in reverse order, leading up to the point where the exception was thrown. It helps developers trace the cause of the exception and identify the line of code where it originated.

Explain the Java Exception Hierarchy.

The Java Exception Hierarchy is a class hierarchy rooted at the Throwable class. This class has two main subclasses:

  • Error: Represents serious problems that a reasonable application should not try to catch. These are often due to system-level issues, like VirtualMachineError.
  • Exception: Represents conditions that a program should handle. The Exception class has a key subclass, RuntimeException, which is the superclass for all unchecked exceptions. All other exceptions inheriting directly from Exception are checked exceptions.

            Object
               |
           Throwable
         /         \
    Error        Exception
                   |
             (Checked Exceptions)
                   |
            RuntimeException
                   |
          (Unchecked Exceptions)

How do you create a custom exception in Java?

To create a custom exception, you simply create a new class that extends either the Exception class (for a checked exception) or the RuntimeException class (for an unchecked exception). It's good practice to provide constructors that call the superclass's constructors to handle the exception message and cause.


public class InvalidAgeException extends Exception {

    public InvalidAgeException(String message) {
        super(message);
    }

    public InvalidAgeException(String message, Throwable cause) {
        super(message, cause);
    }
}

What is the difference between Exception and Error in Java?

  • Exception: Represents a condition that a program should catch and handle. These are recoverable. Examples include IOException and SQLException.
  • Error: Represents a serious problem that a program shouldn't try to handle. They are typically unrecoverable and indicate a serious system-level issue. Examples include OutOfMemoryError and StackOverflowError.

What is the difference between final, finally, and finalize()?

  • final (keyword): A modifier used with classes, methods, and variables. A final class cannot be subclassed, a final method cannot be overridden, and a final variable's value cannot be changed after it's initialized.
  • finally (block): The block of code in a try-catch-finally construct that is guaranteed to execute, regardless of whether an exception is thrown or caught.
  • finalize() (method): A method in the Object class that the garbage collector calls just before an object is garbage-collected. This method is deprecated and should not be relied upon for critical cleanup.

Explain the concept of exception propagation in Java.

Exception propagation is the mechanism by which an exception travels up the call stack if it is not handled in the method where it occurred. When an exception is thrown, the JVM searches for a matching catch block in the current method. If not found, the exception is propagated to the calling method (the one that called the current method). This process continues up the call stack until a suitable catch block is found or the main method is reached. If the exception is not caught even there, the JVM terminates the program and prints the stack trace.

Explain about Throwable class in Java. What are the two main subclasses of the Throwable class in Java? How does the Throwable class facilitate exception handling in Java? What methods are commonly used from the Throwable class to retrieve exception information?

The Throwable class is the root class of the Java exception hierarchy. All exceptions and errors in Java are subclasses of Throwable.

The two main subclasses are:

  • Error: Represents unrecoverable, serious problems.
  • Exception: Represents recoverable conditions that a program should handle.

The Throwable class facilitates exception handling by providing a common base for all exceptions and errors, ensuring that they can be caught and processed uniformly. It also provides essential methods for retrieving information about the exception, which is crucial for debugging and error logging.

Commonly used methods from the Throwable class include:

  • getMessage(): Returns the detailed message string of this throwable.
  • toString(): Returns a short description of the throwable.
  • printStackTrace(): Prints the stack trace to the standard error stream.
  • getCause(): Returns the cause of this throwable, or null if the cause is nonexistent or unknown.

Explain Checked and Unchecked Exceptions in Java with examples.

Checked Exceptions:

These are exceptions that the compiler forces you to handle. They are subclasses of the Exception class (but not RuntimeException). They represent external, predictable problems that a program should gracefully recover from. You must either use a try-catch block or declare them with the throws keyword in the method signature.


// Example of a Checked Exception (IOException)
import java.io.FileReader;
import java.io.IOException;

public class CheckedExample {
    public static void main(String[] args) {
        try {
            FileReader file = new FileReader("file.txt");
            // The compiler forces us to handle this
        } catch (IOException e) {
            System.out.println("File not found or IO error occurred.");
            e.printStackTrace();
        }
    }
}

Unchecked Exceptions:

These exceptions are not checked at compile time and are subclasses of RuntimeException. They typically represent programming errors that should be fixed rather than handled, such as a logical flaw in the code. The compiler does not require you to catch or declare them.


// Example of an Unchecked Exception (ArrayIndexOutOfBoundsException)
public class UncheckedExample {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3};
        // This will throw an exception at runtime
        // The compiler doesn't force a try-catch
        System.out.println(numbers[3]); 
    }
}
