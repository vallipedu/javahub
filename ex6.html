Java Exception Handling

This page demonstrates various aspects of exception handling in Java, including basic try-catch blocks, multiple catch clauses, and the creation of both built-in and user-defined exceptions.

a) Describing Exception Handling Mechanism

This program demonstrates the fundamental try-catch-finally mechanism. The code within the try block is monitored for exceptions. If an exception occurs, the program's control is transferred to the catch block. The finally block will always execute, regardless of whether an exception was caught or not. 


public class BasicExceptionHandling {
    public static void main(String[] args) {
        int numerator = 10;
        int denominator = 0;

        try {
            // This block contains the code that might throw an exception.
            int result = numerator / denominator; // This will throw an ArithmeticException.
            System.out.println("Result: " + result); // This line is skipped.
        } catch (ArithmeticException e) {
            // This block catches the exception and handles it gracefully.
            System.out.println("Error: Cannot divide by zero.");
            System.out.println("Exception details: " + e.getMessage());
        } finally {
            // The finally block always executes, whether an exception occurred or not.
            System.out.println("Execution of finally block.");
        }

        System.out.println("Program continues after the try-catch-finally block.");
    }
}

b) Illustrating Multiple Catch Clauses

A single try block can be followed by multiple catch blocks to handle different types of exceptions. The JVM will execute the first catch block whose exception type matches the thrown exception. 


public class MultipleCatchBlocks {
    public static void main(String[] args) {
        try {
            String str = null;
            // Example 1: NullPointerException
            System.out.println(str.length()); 

            // Example 2: ArithmeticException (uncomment to test this one)
            // int a = 10 / 0;

            // Example 3: ArrayIndexOutOfBoundsException (uncomment to test this one)
            // int[] numbers = {1, 2, 3};
            // System.out.println(numbers[4]);

        } catch (NullPointerException e) {
            System.out.println("Caught NullPointerException: " + e.getMessage());
        } catch (ArithmeticException e) {
            System.out.println("Caught ArithmeticException: " + e.getMessage());
        } catch (Exception e) {
            // A generic catch block to handle any other exception.
            System.out.println("Caught a generic Exception: " + e.getMessage());
        }

        System.out.println("Program finished.");
    }
}

c) Creation of Java Built-in Exceptions

This program demonstrates how a built-in exception, specifically NumberFormatException, is thrown when an operation fails. We'll use Integer.parseInt(), which throws this exception if the string cannot be converted to a number. 


public class BuiltinExceptionDemo {
    public static void main(String[] args) {
        String numString = "abc";

        try {
            int number = Integer.parseInt(numString);
            System.out.println("The number is: " + number);
        } catch (NumberFormatException e) {
            System.out.println("An exception occurred during number conversion.");
            System.out.println("Exception message: " + e.getMessage());
            System.out.println("This is a built-in exception handler.");
        }
    }
}

d) Creation of User Defined Exception

You can create your own custom exception classes to represent specific error conditions in your application. This example defines a custom exception called InvalidAgeException. 


// Step 1: Define a custom exception class.
// It's good practice to extend Exception for checked exceptions, or RuntimeException for unchecked.
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

// Step 2: Use the custom exception in a class.
public class UserDefinedExceptionDemo {

    public static void checkAge(int age) throws InvalidAgeException {
        if (age < 0 || age > 150) {
            throw new InvalidAgeException("Age is not valid: " + age);
        } else {
            System.out.println("Age is valid.");
        }
    }

    public static void main(String[] args) {
        try {
            // Example with a valid age
            checkAge(25); 
            // Example with an invalid age
            checkAge(-5); 
        } catch (InvalidAgeException e) {
            System.out.println("Caught user-defined exception!");
            System.out.println(e.getMessage());
        }
    }
}
