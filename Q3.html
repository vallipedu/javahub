Problem Statement: Account Management System

Your task is to create a very simple banking application using Java. The goal is to manage a single bank account and practice the fundamental concepts of **classes**, **objects**, **encapsulation**, and **exception handling**.

Functional Requirements

  • Account Properties: The system must store an account holder's name and their current balance.
  • Transactions: The account must support two basic operations: a **deposit** to add money and a **withdrawal** to take money out.
  • Validation: The system should prevent transactions with negative amounts.
  • Error Handling: For withdrawals, instead of simply printing a message, the system must use a custom exception to signal that there are not enough funds.

Technical Requirements

  • Create an Account Class: Design a single class named Account.
  • Use Encapsulation:
    • The fields for the account balance and owner's name must be declared as **private**.
    • Create **public methods** like deposit() and withdraw() to control how the balance is changed.
  • Methods with Exception Handling:
    • A constructor to create an Account object with an initial balance and owner name.
    • A deposit(double amount) method that adds the amount to the balance only if the amount is positive.
    • A withdraw(double amount) method that must **throw** a custom exception if the amount is positive but the balance is insufficient. The method signature will need to declare the exception, e.g., public void withdraw(double amount) throws InsufficientFundsException.
  • Create a Custom Exception:
    • Create a new class named InsufficientFundsException that **extends** the `Exception` class. This class can have a simple constructor that calls the parent constructor, e.g., super("Insufficient funds for this transaction.").
  • Demonstrate in Main: In your `main` method, create a single `Account` object. When calling the `withdraw` method, you must enclose the call in a **try-catch** block to handle the potential exception.
